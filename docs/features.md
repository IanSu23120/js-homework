# Features

## 介紹首頁

路由：`/`

訪客狀態：

- 說明產品概念。
- 介紹每日行程、群組協作、地圖推薦與共同記帳。
- 提供登入與註冊入口。

登入狀態：

- 顯示下一趟旅程。
- 顯示旅程、規劃天數、行程項目與群組統計。
- 顯示最近旅程及快速入口。
- 登入成功後返回此會員首頁。

## 功能頁

路由：`/planner`

功能：

- 查看旅程列表。
- 查看旅程統計。
- 新增旅程。
- 從地圖探索推薦景點。
- 將景點加入指定旅程與日期。

## 新增旅程

欄位：

- 旅程名稱
- 目的地城市
- 出發日期
- 回程日期
- 封面顏色

輔助資訊：

- 城市地圖
- 推薦景點
- 天氣預測

若使用者勾選推薦景點，儲存旅程時會將景點加入第一天行程。

## 城市支援

目前支援：

- 日本：東京、大阪、京都
- 韓國：首爾、釜山、濟州
- 台灣：台北、台南、高雄
- 泰國：曼谷、清邁
- 越南：河內、胡志明市、峴港
- 其他：新加坡、香港、澳門、吉隆坡、峇里島、上海

城市資料來源：`src/data/cities.js`

## 推薦景點

推薦景點資料來源：`src/data/attractions.js`

每個景點包含：

- id
- city
- aliases
- name
- category
- description
- lat
- lng

## 旅程詳情

路由：`/trip/:tripId`

功能：

- 切換不同日期。
- 新增自訂行程。
- 編輯既有行程。
- 刪除行程。
- 依行程位置顯示地圖 marker。
- 點選行程或 marker 後同步聚焦。

## 群組分享

已建立的旅程可在旅程卡片調整分享設定：

- 不分享給群組。
- 分享至目前使用者已加入的任一群組。
- 群組成員可以查看與共同編輯共享行程。
- 只有行程建立者可以變更分享設定或刪除旅程。
- 行程備註以留言形式保存，顯示留言者帳號。
- 行程費用顯示實際記帳者帳號。

## 群組提案與投票

共享旅程在 `/trip/:tripId/suggestions` 提供獨立協作提案工作區，入口位於行程頁的「費用管理」旁：

- 群組成員可提出景點、餐廳、交通、住宿或其他建議。
- 可從 Google Places 推薦選取地點，或直接點擊地圖設定位置。
- 提案可附說明，位置由地圖自動記錄，不需手動輸入經緯度。
- 地圖會同時顯示附近推薦與已有位置的群組提案。
- 提案者或群組創立者可編輯、刪除尚未接受的提案。
- 成員可投贊成票，也可取消投票。
- 創立者可選擇旅程日期與時間接受提案。
- 接受提案後，系統會建立正式行程項目並保留地圖位置。
- 已接受的提案不可重複投票或再次加入。
